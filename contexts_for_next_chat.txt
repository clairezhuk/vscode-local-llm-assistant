Project Essence

A local AI programming assistant (similar to GitHub Copilot) built as a VS Code
extension.

  - Resources: Optimized for entry-level hardware (2GB VRAM, 8GB RAM). It
    utilizes quantized models (e.g., Qwen 2.5 Coder 1.5B) via llama.cpp.
  - Key Features: Streaming markdown-supported chat, Fast/Thinking modes,
    context awareness via attached files & workspace detection, code
    autocompletion (Ghost text), and Human-in-the-loop terminal execution.

Class Responsibilities

Frontend (VS Code Extension - TypeScript):

  - chat_view.ts (ChatViewProvider): Manages the chat Webview. Handles streaming
    data consumption via fetch API, processes JSON-chunked responses, manages
    AbortController for stopping generation, and renders interactive UI for
    command confirmation (Accept/Reject).
  - ghost_text.ts (GhostTextProvider): Implements inline autocompletion using
    a 15-line sliding window to maintain low latency and context relevance.
  - client.ts: Handles standard Axios requests for command confirmation and
    completions.

Backend (FastAPI - Python):

  - llm.py (LLMEngine): A wrapper around llama_cpp supporting both atomic
    generation and generator-based streaming (stream=True).
  - orchestrator.py (Orchestrator): The core logic controller. Manages the
    Streaming Generator lifecycle, handles Fast vs Thinking reasoning paths,
    maintains a state for pending commands, and implements code self-correction.
  - planner.py (TaskPlanner): Deconstructs coding tasks into logical JSON-based
    steps for the "Thinking" mode.
  - file_processor.py (FileProcessor): Optimizes context by extracting code
    signatures (AST) or summaries.
  - tools.py (ToolExecutor): Executes terminal commands with workspace awareness
    (cwd support).

Current State

A real-time streaming pipeline is implemented:

1.  Manual Control (UI-driven): Unlike earlier versions, the user now manually
    selects the Intent (Learn/Code/Terminal) and Mode (Fast/Thinking), ensuring
    deterministic behavior.
2.  Fast Mode: A direct pipeline: Preprocessing -> Context Injection ->
    Streaming Generation. Optimized for simple questions and quick snippets.
3.  Thinking Mode (Agentic Workflow):
      - Planning: Uses planner.py to show the user a roadmap of the task.
      - Execution: For Intent 2 (Code), the backend performs a Self-Correction
        cycle (up to 3 attempts). It validates the generated code via ast.parse
        for syntax errors and missing imports. If a failure occurs, it clears
        the UI block and regenerates the code with error feedback.
4.  Human-in-the-loop Execution (Intent 3): Terminal commands are not executed
    automatically. The orchestrator proposes a command, and the Webview renders
    Accept/Reject buttons. Execution only occurs upon user confirmation within
    the project's root directory.
5.  Workspace Awareness: The extension automatically detects the user's active
    workspace path and passes it to the backend to ensure commands (like mkdir
    or npm install) affect the correct files.

Testing System (Benchmarks)

An automated sandbox designed to verify architectural stability.

  - Structure: JSON-based suites for Algorithms, CLI, and Complex Logic.
  - Runner: Executes LLM-generated code in temporary isolated folders, verifying
    logic via assert statements and CLI output.
  - Metrics: Tracks Intent accuracy (now used for manual verification),
    Formatting (Markdown), and Execution success.

Roadmap (Next Steps)

1.  RAG / TF-IDF Search: Implement a lightweight local vector or keyword search
    to pull relevant code snippets from large projects into the prompt,
    bypassing the context window limits of 1.5B models.
2.  Isolated Sandbox Execution: Move from static ast checks to actual runtime
    execution in a restricted environment (e.g., using a separate process or
    container) to catch logical RuntimeError issues.
3.  Advanced Self-Correction: Expand the correction loop to "Thinking" mode's
    individual steps, allowing the model to "test" its own plan as it
    progresses.
4.  UI Traceability: Add a "Thought Trace" toggle to show users the hidden
    system prompts and raw LLM reasoning that led to a specific output.
