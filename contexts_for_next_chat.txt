Project Essence

A local AI programming assistant (similar to GitHub Copilot) built as a VS Code
extension.

  - Resources: Optimized for entry-level hardware (2GB VRAM, 8GB RAM). Utilizes
    quantized Qwen 2.5 Coder 1.5B via llama.cpp з обмеженим контекстом (2048
    n_ctx) та зменшеним розміром батчів для стабільності CUDA.
  - Key Features: Real-time streaming chat, Agentic Thinking workflows,
    concurrency management for shared GPU access, and human-in-the-loop command
    execution.

Class Responsibilities

Frontend (VS Code Extension - TypeScript):

  - chat_view.ts (ChatViewProvider): Manages the Webview UI. Implements
    JSON-stream buffering (захист від розрізаних чанків), передає шлях робочої
    директорії до бекенду та обробляє AbortController для безпечного
    переривання генерації.
  - ghost_text.ts (GhostTextProvider): Забезпечує автодоповнення коду. Працює
    паралельно з чатом через чергу запитів на бекенді.

Backend (FastAPI - Python):

  - llm.py (LLMEngine): Обертка над llama_cpp. Налаштована на thread-safe роботу
    (atomic/streaming). Оптимізована під низьку пам'ять через параметри n_batch
    та n_gpu_layers.
  - orchestrator.py (Orchestrator): Центральний контролер. Керує чергою доступу
    до GPU через asyncio.Lock, запобігаючи Access Violation. Реалізує складні
    цикли самоперевірки та багатоетапного планування.
  - planner.py (TaskPlanner): Генерує кроки для виконання складних задач,
    мінімізуючи "галюцинації" моделі через розбиття контексту.
  - tools.py (ToolExecutor): Виконує команди у контексті workspace_path
    користувача.

Current State

Впроваджено Agentic Reasoning Pipeline, що дозволяє моделі 1.5B вирішувати
складні задачі через самокритику:

1.  Concurrency Safety (GPU Locking): Реалізовано механізм замків
    (asyncio.Lock). Тепер запити від Ghost Text та чату шикуються в чергу, що
    запобігає краху CUDA при одночасному зверненні до пам'яті відеокарти.
2.  Fast Mode: Пряма генерація відповіді для простих питань.
3.  Thinking Mode (Recursive Chain-of-Thought):
      - Goal Analysis: Модель спочатку формулює стислу мету запиту, щоб тримати
        "фокус" протягом усього циклу.
      - Iterative Critique: Кожна відповідь перевіряється (логічно або через
        ast.parse). Якщо результат не задовольняє мету, причина помилки
        повертається в наступну ітерацію (до 3 спроб).
      - Code Assembly: Для Intent 2 (Code) система генерує фрагменти за планом,
        а окремий етап "Архітектора" збирає їх у фінальний файл, усуваючи
        дублювання імпортів та функцій.
4.  Robust Streaming: Фронтенд використовує текстовий буфер для збирання
    розірваних JSON-пакетів, що забезпечує плавне відображення тексту
    навіть при нестабільній швидкості генерації.
5.  Safe Terminal: Команди генеруються з урахуванням технічних обмежень
    середовища, відображаються користувачу для перевірки та виконуються
    строго в робочій директорії проекту.

Testing System (Benchmarks)

Автоматизована пісочниця для перевірки стабільності архітектури.

  - Structure: Тематичні блоки (Алгоритми, CLI, Логіка).
  - Runner: Створює ізольовані temp-директорії, виконує згенерований код та
    перевіряє результат через assert.
  - Current Metric: Фокус зміщений на успішність проходження циклу
    самовиправлення (Success Rate після ретраїв).

Technical Constraints & Solutions

  - VRAM Limit (2GB): Вирішено через зменшення n_ctx до 2048 та n_batch до 128.
    Це дозволяє уникнути out of memory при глибоких роздумах.
  - Thread Safety: Вирішено через asyncio.to_thread та глобальний asyncio.Lock в
    межах об'єкта Orchestrator, що робить локальний сервер стійким до
    паралельних запитів від VS Code.
